ASR STUDIO - COMPLETE TECHNICAL DOCUMENTATION
=============================================

1. OVERVIEW
-----------
ASR Studio is a Flask application for speech-to-text and document intelligence workflows with:
- media ingestion (audio/video)
- transcription (faster-whisper)
- speaker diarization (pyannote)
- speaker mapping and smoothing
- summary generation (Ollama or BART)
- translation (NLLB)
- live chunk transcription
- session history persistence (MongoDB)
- document ingestion + RAG Q&A (MongoDB)


2. RUNTIME FLOW
---------------
Startup sequence:
1. Flask app created.
2. Workspace folders ensured:
   audio/, videos/, outputs/, documents/, recordings/
3. ASR model loads (preferred ASR_MODEL_SIZE; fallback to "medium").
4. Diarization pipeline loads.
5. MongoDB init + indexes.
6. HTTP routes are served.

Primary media flow (/process):
1. Accept uploaded media or path payload.
2. If video, extract audio.
3. Ensure WAV + workspace copy.
4. Try Demucs enhancement (if enabled); fallback to original on error.
5. Transcribe audio segments with timestamps and words.
6. Run diarization.
7. Map words/segments to speakers.
8. Persist session history to MongoDB.
9. Return transcript + metadata.

Document flow (/process_document):
1. Accept uploaded document (.pdf/.docx/.txt).
2. Extract text (PDF -> pypdf; optional OCR for scanned PDFs).
3. Summarize document (non-meeting format).
4. Chunk text (recursive splitter) + embed with sentence-transformers.
5. Store document metadata in document_store and chunks in document_chunks.
6. Return document summary + document_id for RAG Q&A.

RAG Q&A flow (/api/document/ask):
1. Retrieve top-k chunks by cosine similarity.
2. Build prompt with context + conversation history.
3. Generate answer via Ollama.
4. Persist chat history for the document.


3. PROJECT STRUCTURE
--------------------
main.py
- Flask app + all routes

- faster-whisper model loading
- transcription logic with retry scoring strategy
- live chunk transcription

app/diarization.py
- pyannote pipeline loading
- diarization execution and optional waveform preload

app/mapper.py
- speaker mapping algorithm
- overlap scoring, nearest fallback, smoothing, merge logic

app/media_utils.py
- media conversion, extraction, Demucs enhancement
- document text extraction (PDF/DOCX/TXT)

app/summarize.py
- summary backend integration (Ollama/BART)
- meeting-minutes for transcripts
- plain summaries for documents

app/translation.py
- NLLB loading (offline-first)
- translation API methods and language code resolution

app/processing_service.py
- orchestration for media/document processing
- persistence helpers

app/history_store.py
- session history CRUD (MongoDB)

app/document_store.py
- document metadata CRUD (MongoDB)

app/document_chunks.py
- document chunk storage (MongoDB)

app/document_rag.py
- chunking, embeddings, retrieval, history-aware Q&A

app/semantic_search.py
- semantic search for transcript history

app/db.py
- MongoDB connection and index creation

templates/index.html + static/js/index.js + static/css/index.css
- web UI
- recording controls
- translation dropdown auto-translation behavior
- history rendering
- export actions
- document Q&A and preview

scripts/download_nllb_model.py
- utility script to download NLLB locally and generate env file


4. API ROUTES
-------------
GET /
- returns web UI (index.html)

GET /audio/<filename>
GET /videos/<filename>
GET /documents/<filename>
- static file serving for previews/playback

POST /process
Input:
- multipart/form-data: audio_file=<file>
  OR
- JSON: {"file_path":"...", "filename":"..."}
Output:
- {
    "session_id": str,
    "transcript": [ {speaker,start,end,text}, ... ],
    "summary": str,
    "processed_file": str,
    "before_audio_file": str,
    "after_audio_file": str,
    "source_video": str
  }

POST /transcribe_chunk
Input:
- multipart/form-data: audio_chunk=<blob>
Output:
- {"text": "chunk transcription"}

POST /summarize_text
Input JSON:
- content (required)
- meeting_title (required)
- meeting_date (required)
- meeting_place (required)
- session_id (optional, for persistence)
Output:
- {"summary": "..."}

POST /process_document
Input form-data:
- document_file (required)
Output:
- {
    "summary": str,
    "document_filename": str,
    "document_type": str,
    "document_text": str,
    "document_id": str,
    "chunk_count": int
  }

POST /api/document/ask
Input JSON:
- document_id (required)
- question (required)
- top_k (optional)
Output:
- {"answer": "...", "sources": [...], "history": [...]}

GET /api/documents
Output:
- {"documents": [document_entry, ...]}

GET /api/documents/<doc_id>
Output:
- {"document_id":..., "filename":..., "document_type":..., "text_preview":..., "summary":..., "chat_history":...}

POST /translate
Input JSON:
- target_lang (required)
- source_lang (optional)
- text (string) OR texts (array of strings)
Output for text:
- {"text": "...", "target_lang":"...", "source_lang":"..."}
Output for texts:
- {"texts":[...], "target_lang":"...", "source_lang":"..."}

GET /history
Output:
- {"history": [history_entry, ...]}

GET /history/<session_id>
Output:
- full session payload with transcript + summary

POST /history/<session_id>/transcript
Input JSON:
- transcript (optional list)
- summary (optional string)
Output:
- {"ok": true}

PATCH /history/<session_id>
Input JSON:
- title (required)
Output:
- {"ok": true, "title": "..."}

DELETE /history/<session_id>
Output:
- {"ok": true}


5. DATA SCHEMAS (MongoDB)
-------------------------
Collections:
- history
- document_store
- document_chunks

History (history):
- session_id
- title
- processed_file
- before_audio_file
- after_audio_file
- source_video
- transcript: list of segments {speaker, start, end, text}
- summary
- owner
- created_at, updated_at

Document metadata (document_store):
- document_id
- filename
- document_type
- text_preview
- summary
- chunk_count
- chat_history
- owner
- created_at, updated_at

Document chunks (document_chunks):
- document_id
- index
- text
- embedding


6. ENVIRONMENT VARIABLES
------------------------
Flask:
- FLASK_DEBUG (default: 1)
- FLASK_USE_RELOADER (default: 0)

MongoDB:
- MONGODB_URI (default: mongodb://localhost:27017)
- MONGODB_DB (default: asr_studio)

ASR:
- ASR_MODEL_SIZE (default: medium)

Diarization:
- HUGGINGFACE_TOKEN

Summarization:
- SUMMARY_BACKEND (ollama | bart)
- SUMMARY_MODEL (ollama model name, default: mistral)
- OLLAMA_URL (default: http://localhost:11434/api/generate)

Document OCR:
- DOCUMENT_OCR (1/true/yes/on to enable OCR for PDFs)
- DOCUMENT_OCR_LANGS (default: eng)

RAG:
- RAG_MODEL (default: mistral)
- DOC_CHUNK_SIZE (default: 2000)
- DOC_CHUNK_OVERLAP (default: 80)
- DOC_RETRIEVE_TOP_K (default: 5)

Translation (NLLB):
- NLLB_MODEL_PATH (optional)


7. TROUBLESHOOTING
------------------
- MongoDB connection refused: start MongoDB or set MONGODB_URI.
- Document too large errors: resolved by chunk storage in document_chunks. Ensure MongoDB is running and indexes are initialized.
- Scanned PDFs return no text: enable OCR and install dependencies.
- Ollama summary/Q&A not responding: verify OLLAMA_URL and model availability.
