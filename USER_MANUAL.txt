AI KNOWLEDGE STUDIO - USER MANUAL
================================

1) START THE APP
----------------
1. Install dependencies:
   pip install -r requirements.txt

2. Start backend:
   python main.py

3. Open browser:
   http://127.0.0.1:5000


2) WHAT THE APP CAN DO
----------------------
This application supports:
- plain chat in the docked assistant bar
- audio/video transcription with speaker diarization
- live meeting transcription
- transcript summarization (minutes format)
- document upload and document Q&A
- transcript Q&A
- transcript search
- translation
- text-to-speech
- session and document history


3) TWO WAYS TO USE THE APP
--------------------------
A. Plain chat mode
- Type into the assistant bar without uploading a file.
- The assistant replies like a chatbot.
- Follow-up chat turns keep recent conversation history in the current session.

B. Upload + query mode
- Attach a file using the upload button in the assistant bar.
- The file is staged first.
- Nothing is processed yet.
- Then type your instruction and send it.
- The app runs the matching workflow based on your query.

5) PLAIN CHAT MODE
------------------
Use the assistant bar without uploading a file.

Examples:
- What day is today?
- What is 2+2?
- Explain what this app can do.
- How do I upload a document?

Notes:
- The chat keeps short-term conversation history in the current browser session.
- Date/time questions are answered from runtime code.
- Many math questions are solved deterministically.


6) MEDIA PROCESSING
-------------------
To process audio or video:
1. Attach an audio/video file.
2. Type a query like:
   - transcribe this recording
   - transcribe and diarize this audio
   - summarize this meeting after transcription
3. Send the query.

What happens:
- media is prepared
- transcription runs
- speaker diarization runs
- transcript is shown with timestamps and speakers
- audio/video preview appears when available


7) LIVE MEETING MODE
--------------------
Buttons:
- Start live Meeting
- Pause
- Stop

How it works:
- microphone audio is captured in chunks
- chunks are transcribed live
- live text appears in the interface
- when recording ends, the result can be staged for agent-driven follow-up processing


8) TRANSCRIPT Q&A
-----------------
Once a transcript session exists, you can:
- ask questions about the transcript (answers appear inside the Q&A panel)
- search for relevant parts
- translate transcript-derived content
- convert transcript-derived content to speech

Examples:
- What did the client say about deadlines?
- Search this transcript for pricing discussion.


9) DOCUMENT MODE
----------------
To use document workflows:
1. Attach a PDF, DOCX, or TXT file.
2. Type a query such as:
   - summarize this document
   - ask this document about the conclusion
   - what are the key findings in this document?
3. Send the query.

What happens:
- text is extracted
- document summary is generated
- the document is indexed for question answering
- document history appears in the sidebar

Notes:
- PDF mini preview is available from the sidebar
- document Q&A is history-aware for that document thread


10) SUMMARIZATION
-----------------
Transcript summaries:
- use the top Summarize button or a query-driven request
- transcript summaries use meeting-style formatting

Document summaries:
- use plain summary formatting, not meeting minutes


11) TRANSLATION
---------------
Use the translation language dropdown for supported translation flows.

You can:
- translate summaries
- translate transcript-derived text
- translate document-derived text
- use query-first commands such as:
  - translate the summary to Hindi
  - translate this answer to Tamil


12) TEXT-TO-SPEECH
------------------
The app can convert supported text into speech.

Examples:
- convert this summary to speech
- read this answer aloud

Notes:
- generated audio is stored in audio/
- gTTS usually needs internet access
- if gTTS fails, the browser falls back to SpeechSynthesis


13) HISTORY
-----------
Transcript History sidebar:
- open an older transcript session
- rename session
- delete session

Document History sidebar:
- open an older document
- refresh list
- clear list

New Session button:
- refreshes the app and returns to the home screen
- does not log out

Hard reload (Ctrl+Shift+R):
- triggers logout by design


14) EXPORT
----------
- Export Transcript -> downloads transcript as **PDF** in minutes format
- Export Summary -> downloads summary as **PDF** in minutes format

2. Then ask:
   What is my name and which team do I work in?

Expected behavior:

18) COMMON ISSUES
-----------------
Issue:
MongoDB connection refused
Fix:
- start MongoDB
- or set MONGODB_URI

Issue:
Ollama-backed chat / summary / RAG not responding
Fix:
- ensure Ollama is running
- verify OLLAMA_URL
- verify the configured model exists

Issue:
NLLB model not found offline
Fix:
- set NLLB_MODEL_PATH
- or place the model in expected local folders

Issue:
Diarization model not loading
Fix:
- set HUGGINGFACE_TOKEN

Issue:
NeMo diarization VAD tensor error
Fix:
- ensure input audio is mono 16k (pipeline forces mono conversion)

Issue:
Live transcription not working
Fix:
- allow microphone access
- check MediaRecorder browser support

Issue:
Scanned PDF returns no text
Fix:
- enable OCR with DOCUMENT_OCR=1
- install pdf2image, pytesseract, and system Tesseract
